会创建和‘src’并列的文夹‘bin’,并在‘bin‘下生成二进制可执行文件 ‘hello’

这里为什么没有生成‘pkg’文件夹呢？因为 go install 会依次查找所有GOPATH中的目录寻找hello包和它依赖包。然后会将报名为main的包生成二进制文件放到GOBIN目录下。将非main包编译成.a文件放到项目对于的pkg目录下。而我们的hello.go并没有依赖包。
